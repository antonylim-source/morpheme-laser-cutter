화면 비율 관련하여 다음을 적용해 줘. 고정 크기 1280×800의 #stage를 만들고, 브라우저 크기에 맞춰 비율 유지로 스케일만 조정해 화면에 표시하세요. 활동화면 정렬은 top=0, 가로 중앙입니다.
스케일 계산은 scale = min(window.innerWidth/1280, window.innerHeight/800)을 사용하고,
transform-origin: top center로 설정하세요. 내부 레이아웃은 반응형이 아니며, 스케일만 변합니다.
스크롤은 기본적으로 숨기되, 필요 시만 보이도록 해주세요. 즉, 브라우저 창 크기를 조절해도 앱 전체가 1280x800 비율을 유지하며 자동으로 확대/축소되어야 하며 이로 인해 어떤 화면 크기에서도 레이아웃이 깨지지 않고 최적의 화면을 제공해야 합니다. 


[화면 비율 프롬프트]
HTML, CSS, JavaScript를 사용하여 다음 명세에 따라 웹 페이지를 구현해 주세요.

핵심 요구사항
고정된 1280x800 크기의 콘텐츠 영역(#stage)을 생성합니다. 이 영역은 브라우저 창 크기가 변경될 때마다 가로세로 비율을 유지한 채로 자동 스케일 조정되어 항상 창 안에 꽉 차게 표시되어야 합니다.

세부 명세
HTML 구조

<body> 태그 안에 ID가 stage인 <div> 요소 하나만 포함해 주세요.

#stage 내부에 스케일링을 확인할 수 있도록 간단한 텍스트나 자식 div를 추가해 주세요.

CSS 스타일링

body:

기본 margin을 제거하고, 스크롤바가 생기지 않도록 overflow: hidden;을 적용하세요.

#stage:

너비와 높이를 각각 1280px, 800px로 고정하세요.

화면 상단(top: 0;), 가로 중앙에 위치하도록 정렬하세요.

스케일링의 기준점이 상단 중앙이 되도록 transform-origin: top center;를 설정하세요.

JavaScript 기능

페이지가 로드될 때와 브라우저 창 크기가 조절될 때마다(resize 이벤트) 다음 로직을 실행하는 함수를 작성하세요.

스케일 계산: 스케일 값은 다음 공식을 사용해 계산합니다.

JavaScript

const scale = Math.min(window.innerWidth / 1280, window.innerHeight / 800);
스케일 적용: 계산된 scale 값을 #stage 요소의 transform 속성에 적용하여 크기를 조정하세요. 이때, 가로 중앙 정렬이 유지되어야 합니다.

최종 목표: 사용자가 브라우저 창 크기를 어떻게 조절하든, 1280x800 비율의 #stage가 잘리거나 레이아웃이 깨지는 일 없이 항상 화면에 최적화된 크기로 표시되어야 합니다.
